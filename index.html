What is node.js?

A javascript platform

Can build lightweight web applications

No appache needed!

What is a web server?

  • Handles web requests and routes them to appropriate pages

  • Stores additional application data

    • databases

What is front-end & back-end?

Back-End or Server-Side      

What does that mean though?

It's complicated
i/o=input/output

Imagine facebook on a terrible server

When looking up you’re news feed in the database, it just stalls because the database takes too long

Most servers do that! They wait for I/O to finish


Event driven server side i/o


Event driven server side i/o


  • Other web frameworks can do this, but

    • with a lot of setup and coding and thread management

  • Node.js does it all for you!
  • And can build scalable applications quickly

When should you use node.js?

  • Great for real-time applications:
    • online games, collaboration tools, …
    • anything quick and event driven

WHEN SHOULD YOU NOT USE NODE.JS?

  • Your server request is dependent on heavy CPU consuming algorithm/job

Step 1 - INSTALL NODE.JS

nodejs.org


STEP 2 – INSTALL EXPRESS

  • A simple web application framework for node.js
  • Launch Terminal or Command Prompt
  • Run the following command (you can copy text) 

							$ sudo npm install -g express

Windows users might have to first run:  

cd C:\Program Files\nodejs

STEP 3 – Create an express project

							$ cd ~/Documents/
$ mkdir nodeProjects
$ cd nodeProjects
$ express nodetest1

Windows users, if cd ~/Documents gives errors, run 

cd C:\Users\[your user name]\Documents

STEP 4 – Install Dependencies

							$ cd nodetest1/
$ npm install

STEP 5 – Run Your First Node Web App

							$ node app.js

Windows users might get a prompt to allow access

STEP 9 - INSTALL SUPERVISOR!

  • to avoid constant restarting of node with changes, run (CTRL+C in terminal) then run:

							$ supervisor node app.js
